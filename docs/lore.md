# ハノーバ世界観 doctrine（lore 正本）

このファイルは Hanōba の世界観・市長キャラ・市民レベル・本（市民手帳）のページモデルの
**正本（doctrine）**である。表示テキストの実体は `src/lib/lore/cityHall.ts`（UI が描画する単一ソース）に
あり、こちらは「何を・なぜそう書くか」を人間向けに定める。文言を足すときは cityHall.ts を、
方針を変えるときはこのファイルを更新する。

関連 Issue: #163（本・市民レベル）、#469（図鑑化・1レベル=1ページ・街の地図を早期ご褒美ページに）。
市長ブログ等の運用は #164、地図の対話版は #137、言語別は #147。

## 市長キャラ・バイブル（ボタニクス・フォン・ハノーバ）

ハノーバ市の初代にして現市長。架空都市の NPC。手帳・市政文書・将来のブログはこの声で書く。

- **一人称**: 「市長」（自称は「市長」で通す。「わたくし」は未使用＝実コピーに合わせる）
- **二人称**: 「諸君」
- **口癖**: 「おっほん」の咳払いから入ることが多い
- **態度**: 大仰・気取り（フォン＝貴族気取り）。丁寧だが独断的。余裕がある。
- **思想**: 全肯定。批判から入らない。特定の植物・人を貶めない。暴論は**全部「礼賛／擁護」の方向**へ振る
  （例: 「区画は正方形に限る。なぜか？ 美しいからだ。理由は以上である」＝独断だが誰も傷つけない）。
- **信条**: 植物に意志が宿ると本気で信じている。育てる意志を「地代」と呼ぶ。

### NG（やってはならない）

- 優劣で貶す／序列をつけて見下す
- 人物の中傷
- 実在の作品・人物・サービスへの攻撃
- 生成画像（市長の立ち絵等を AI 生成で出さない）

## 市民レベル（Nostr 由来＝backendless）

会員ランクをサーバに持たない。その場の Nostr 由来の手がかりだけからレベルを決める純ロジック
（`src/lib/lore/citizen.ts`）。閾値は `TENURE_POSTS = 5` / `TENURE_DAYS = 14`。

図鑑化（#469）で **1 レベル=1 ページ解放**。手帳のページ解放は L3（条文）で頭打ち（`citizenLevel` cap=3）。
表示ラベルの Ln（`citizenLevelLabel`）は別軸で L4 以降も伸びる（手帳の解放には効かない）。

| レベル | 名称 | 条件 | 開けるページ |
|---|---|---|---|
| L0 | 旅人 | 表示名が未登録（まだ移住していない） | 1p のみ |
| L1 | 市民 | 表示名が登録済み（= 移住完了） | 2p 街の地図まで |
| L2 | 市民L2 | 表示名あり ＋ 投稿数 ≥ 5 ＋ 最古投稿が 14 日以上前 | 3p 沿革まで |
| L3 | 市民L3 | 表示名あり ＋ 投稿数 ≥ 15 ＋ 最古投稿が 30 日以上前 | 4p 市の条文まで |

- 手がかりの取得元: 表示名 = `getDisplayName()`（kind:0 / localStorage の控え）、投稿数・最古投稿 =
  `fetchMyPosts(pubkey)`（t:hanoba の自分の投稿）の件数と最小 `createdAt`。
- レベル判定は純関数 `citizenLevel({ hasName, postCount, earliestCreatedAt, now })`。`now` は
  外から秒で渡す（`Date.now` を内部で呼ばない＝fixture テスト可能）。
- **resilient**: 名前があるのに投稿取得が失敗したら L1 にフォールバックする（名乗った市民を締め出さない）。
- 既定ページ: `defaultPage(level)` = L0→1 / それ以外→2（街の地図＝早期のご褒美ページを先に見せる）。
  奥のページ（沿革・条文）は自動では開かず、前送りで辿る。判定中は安全側の 1p を見せ、ロック状態を実市民にちらつかせない。

## 本（市民手帳）のページモデル

在世タイトル＝「ハノーバ市民手帳」。手帳は **図鑑（集めて埋める読み物・1 レベル=1 ページ解放）** に割り切る（#469）。
**タイトルはレベル番号表記で進捗を一目で示す**（#469 変更A・kako-jun 確定）: L1+ は「ハノーバ市民手帳 L{n}」、
L0（旅人・未名乗り）はレベル番号を出さず素のタイトル＋副題「旅人」（`citizen.level.traveler`）。曖昧な副題
（旧 `levelSubtitle`＝ようこそ/市民の手引き/古参の手引き）は廃止。**n は真レベル＝`citizenLevelFull`（非キャップ・L1〜L6）**＝
CitizenStats と同じ表記で、ページ解放用の `citizenLevel`（capped 3）とは別軸（ヘッダの about ラベルにはレベル番号を出さない・#262）。

**市長の言葉は全ページの冒頭に必須**（#469 変更B・kako-jun）: 各ページは「`MayorMark`（語り手アイコン）→ 市長の前口上（lead）→ 本体」で
揃える。welcome は歓迎の辞、map は lead、chronicle/ordinances も冒頭に lead（`cityHall.chronicle.lead` / `cityHall.ordinance.lead`）を持つ。

**機能導線（メニュー）は手帳が持たない**: discover/ranking/me/compose はヘッダ/フッタ（`SiteHeader.astro`/`SiteFooter.astro`）が
既に提供する。手帳が「メニュー（機能導線）」と「読み物（ロア）」を兼ねていたのが密度・発見性問題の正体だったので、
重複メニューを外し、手帳はロアと早期ご褒美（街の地図）に専念する。

- **前方ロック／後方オープン**: 解放済みページ（番号 ≤ `maxUnlockedPage(level)`）には自由に行き来できる。
  その先の 1 枚だけ「？？？」ティザーとして覗ける（枠は見えるが中身は開けない・図鑑式＝進む動機）。
  ティザーのさらに先へは進めない。
- **ページ割付**（順序固定・図鑑順）:
  - p1 移住案内（市長の歓迎の辞・L0）— 新規者向けに `site` の一言説明（正方形・一言・植物だけ）を小さく併記
  - p2 街の地図（L1・早期のご褒美ページ）— 名所（ランドマーク＝葉脈川・正方形の区画・市役所）を読み物にし、
    末尾に「市政の窓口」strip を添える（下表）。地図はまだ描きかけ、の注記つき
  - p3 沿革（年表・遊び・L2）
  - p4 市の条文（ハノーバ市憲章・L3）
- **maxUnlocked**: L0→1 / L1→2 / L2→3 / L3→4（1 レベル=1 ページ）。
- **昇格の味付け**（レベル別に一度だけ・入口で添える）:
  - 市民歓迎「移住、確かに受理した。…」は **L1 が 2p（街の地図）を開いたときだけ**。L2 以上には
    再掲しない（長く居る市民に毎回「移住を受理した」と告げない）。
  - 古参歓迎「おっほん。諸君はもう、市の古い友人だ。…」は **L2 以上が初めて奥（3p 沿革＝古参の最初のページ）に
    達したときだけ**。2p では味付けを出さない（L3 まで解放が伸びても古参歓迎は維持）。

## 架空条文（ハノーバ市憲章）

osaka-kenpo 作法（条文＋市長解説）に倣う。各条は短い条文と、市長の一人称解説をセットで持つ。
以下を正本とする（表示の実体は cityHall.ts）。

- **第一条（土地）** ハノーバ市の土地代は、植物を育てている限り、無料とする。
  - 解説「おっほん。育てる意志こそが地代だ。水をやり、葉を見つめる——それで諸君は十分に納めている。」
- **第二条（区画）** 一つの区画には、正方形の写真一枚と、ひとことを添えるものとする。
  - 解説「区画は正方形に限る。なぜか？ 美しいからだ。理由は以上である。」
- **第三条（住民）** 名を名乗った者を、市民とみなす。
  - 解説「名乗りは移住届だ。本名である必要はない。諸君が諸君であればよい。」
- **第四条（平等）** 市は、すべての植物を等しく愛する。雑草という名の植物は、当市には存在しない。
  - 解説「日陰の苔も、軒先の多肉も、みな市の宝。優劣をつける条例は、永久に制定しない。」
- **第五条（写真）** 当市に掲げる写真は、植物のものに限る。
  - 解説「人もペットも、それぞれの街で輝けばよい。ここは植物の市だ。」

## 実装メモ

- 表示テキストの単一ソース: `src/lib/lore/cityHall.ts`（このファイルは doctrine 正本）。
- 市民レベルの純ロジック: `src/lib/lore/citizen.ts`（fixture テスト `citizen.test.ts`）。
- UI: `src/components/lore/CityHallBook.tsx`（`about.astro` に `client:only="react"` でマウント）。
  `client:only` なのは en→ja 言語フラッシュを断つため（#479）。`client:load` だと殻が既定言語 en で
  SSR され、その英語 HTML が hydrate 前に先に描画される＝`useLayoutEffect` では消せない。島を
  サーバで焼かないことで最初のペイントが閲覧言語になり、マウント時に `.ha-rise` でじわーっと出る。
- 初回の表示言語 auto-detect（#482）: `hanoba:lang` が**未保存 or 無効値**なら `navigator.languages` を走査し、
  対応言語（`LOCALES`＝ja/en/zh/es）に一致する最初のものを既定表示にする（日本語 OS/ブラウザなら初回から ja）。
  検出ロジックは島側 `src/lib/i18n/clientLocale.ts#resolveClientLocale/detectClientLocale` と殻側 `MainLayout.astro`
  の言語 swap（is:inline）の**2箇所で同一契約に保つ**（有効な保存値→使用／それ以外→検出。食い違うとフラッシュ／不一致）。
  言語 swap は実効言語を `document.documentElement.lang` に反映し、**後段の市民ラベル swap はそれを読む**（localStorage
  を再読みせず解決を一本化＝市民手帳ラベルだけ言語が割れるのを防ぐ）。検出値は localStorage に書かない（Twitter
  モデル維持＝保存は `setClientLocale` のユーザー操作だけ）。
- relay 呼び出しは `src/lib/nostr/client.ts` のみ（`fetchMyPosts` を再利用）。
- 言語別（JA/EN）は #147 前提で defer（まず JA）。市長 NPC の手動運用は #164。
